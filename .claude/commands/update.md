Синхронизируй задачи из Plane в роадмапы. Делай строго так:

## Источник
Задачи тянутся **напрямую из Plane Cloud API** скриптом
`scripts/fetch-plane-tasks.mjs` (нужны `PLANE_API_KEY`, `PLANE_WORKSPACE_SLUG`,
`PLANE_PROJECT_ID` в `.env`). Никакого ручного сохранения HTML больше не нужно.

Цель:
- `C:\Users\XBIT-2025\Desktop\TASK-MANAGMEN\roadmaps\muhammad.roadmap.md` — серия **A**
- `C:\Users\XBIT-2025\Desktop\TASK-MANAGMEN\roadmaps\soleh.roadmap.md` — серия **B**

## Шаги
1. Запусти `node scripts/fetch-plane-tasks.mjs`. Он вернёт список задач по одной
   в строке в формате `A1.2 — Название` / `B4.1 — Название` (код серии + ` — ` +
   имя). Скрипт уже отфильтровал мусор и оставил только настоящие задачи.
   - Если скрипт упал с «нет PLANE_API_KEY» — попроси пользователя вписать токен
     в `.env` (см. `.env.example`) и останови синхронизацию.
   - Если вернулось 0 задач — сообщи пользователю, что в Plane нет задач в
     формате `A1.2 — Текст`, и останови синхронизацию (роадмапы не трогай).

2. Сравни найденные коды с тем, что уже есть в роадмапах:
   - Задачи **A** → `muhammad.roadmap.md`, задачи **B** → `soleh.roadmap.md`.
   - Добавляй ТОЛЬКО те коды, которых ещё нет в целевом файле. Существующие не трогай
     (не меняй их статус `[ ]`/`[~]`/`[x]` и текст).

3. Для каждой новой задачи создай аккуратную запись в стиле существующих в роадмапе:
   - Заголовок `### [ ] КОД — Название`
   - Обогати деталями из `CLAUDE.md` (соответствующие компоненты/папки, эндпоинты,
     дизайн-токены, зависимости). **Не выдумывай задачи и не добавляй то, чего нет
     в экспорте** — только обогащай уже существующие в HTML.
   - Группируй по фазам (B0/B4/A0/A1…). Если фазы для кода ещё нет — заведи новую
     секцию `## Фаза <код> — <название>` в логичном месте, перед блоком
     «Контракт с …» в конце файла.

4. Выведи краткий итог: какие коды добавлены, в какой файл, и предупреди, если в
   нумерации серий есть пропуски (например, есть B0 и B4, но нет B1–B3).

## Правила
- Источник истины по задачам — только вывод `scripts/fetch-plane-tasks.mjs`
  (Plane API). Ничего не придумывать.
- Источник истины по деталям (эндпоинты, дизайн, структура) — `CLAUDE.md`.
- A → muhammad, B → soleh. Не перемешивать.
- Не менять уже существующие записи и их статусы.
